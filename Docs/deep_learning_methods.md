# deep_learning_methods.md

## Вкладка «Deep Learning»

Вкладка предназначена для сегментации изображений с помощью предварительно обученных моделей глубокого обучения. Пользователь загружает модель (из файла или выбирает предустановленную архитектуру), применяет её к текущему изображению, получает бинарную маску, которая затем может быть обработана морфологией, преобразована в объекты (контуры, прямоугольники, OBB) и сохранена в YOLO‑формате. Поддерживаются модели U‑Net, DeepLabV3+, YOLO‑seg (любые версии YOLO для сегментации), а также пользовательские ONNX‑модели.

### Основные возможности

1. **Загрузка изображений** – стандартная (кнопки «Load Images», «Load Folder»), с опциональным ресайзом до 640 пикселей (`max_side=640`).

2. **Выбор модели** (выпадающий список):
   - **U‑Net** – архитектура с энкодером-декодером (поддерживаются энкодеры: `resnet18`, `resnet34`, `resnet50`, `efficientnet-b0`, `vgg16`). Требуется файл весов `.pth` и рядом с ним файл метаданных `*_meta.json` (содержит `encoder`, `input_size`, `num_classes`).
   - **DeepLabV3+** – модель из `torchvision`, поддерживает предустановленные бэкбоны (`resnet50`, `resnet101`, `mobilenet_v3_large`). Веса загружаются из PyTorch (ImageNet), файл `.pth` не требуется.
   - **SegFormer** – заявлен, но **не реализован** (показывает предупреждение).
   - **SAM** (Segment Anything Model) – заявлен, но **не реализован**.
   - **YOLO-seg** – любая версия YOLO для сегментации (YOLOv8-seg, YOLOv11-seg и т.д.). Требуется файл `.pt`.
   - **Custom ONNX** – пользовательская модель в формате ONNX.

3. **Настройки модели** (появляются при выборе соответствующей модели):
   - **U‑Net**: путь к файлу `.pth`, энкодер (заблокирован, считывается из метафайла), размер входа (заблокирован), **порог бинаризации** (0.01…0.99, по умолч. 0.5). Все поля, кроме пути и порога, определяются метаданными и не редактируются.
   - **DeepLabV3+**: путь к файлу (не используется, так как веса предобученные), выбор бэкбона (`resnet50`, `resnet101`, `mobilenet_v3`), output stride (8 или 16). Поля заблокированы, но отображаются для информации.
   - **YOLO-seg**: путь к `.pt`; параметры инференса: порог уверенности (0.01–1.0), порог IoU для NMS (0.01–1.0), размер входа (32–1280), чекбокс «Сохранять результаты» (сохраняет предсказания YOLO на диск).
   - **Custom ONNX**: путь к `.onnx` файлу.

4. **Общий выбор устройства**:
   - `Auto (best available)` – автоматически выбирает CUDA, MPS или CPU.
   - `CPU` – принудительно центральный процессор.
   - `CUDA (GPU)` – если доступна видеокарта NVIDIA.
   - `MPS (Apple Silicon)` – для Mac с чипами M1/M2.

5. **Выполнение сегментации**:
   - Кнопка **Segment Current Image** – применяет загруженную модель к текущему изображению.
   - Во время выполнения кнопка блокируется, отображается прогресс‑бар (значение 0→100% по завершении, детального прогресса нет).
   - Результат – бинарная маска (пиксели объекта = 255, фон = 0) сохраняется в `self.prediction_mask` и отображается в окне «DL маска».

6. **Постобработка маски** (аналогично другим вкладкам):
   - **Closing factor**, **Opening factor** (0…0.10), **Kernel shape** (Rectangle, Ellipse, Cross).
   - **Invert mask** – инвертирует итоговую маску после морфологии.

7. **Визуализация (4 окна)**:
   - **Оригинал** – исходное цветное изображение.
   - **DL маска** – результат прямого применения модели (без морфологии).
   - **Морфология** – маска после закрытия/открытия и инверсии.
   - **Аннотированное** – исходное изображение, где фон заменён на белый, а объекты выделены в соответствии с выбранным режимом отрисовки.

8. **Выделение и отрисовка объектов**:
   - **Draw objects** – режим отрисовки: `None`, `Segmentation (Polygon)`, `Bounding Box (Detect)`, `OBB (Oriented Box)`.
   - **Use convex hull** – для полигонов и OBB строить выпуклую оболочку.
   - Список объектов с чекбоксами отображается справа; выбранные объекты подсвечиваются.
   - Координаты выбранных объектов выводятся в текстовое поле (для `detect` – пиксельные, для `obb`/`segment` – нормализованные).

9. **Сохранение аннотаций** – кнопка «Save Labels» сохраняет текущий список объектов в YOLO‑формат (`.txt` рядом с изображением).

10. **Пресеты** – сохраняют и загружают полную конфигурацию: тип модели, путь к модели (если есть), параметры инференса (YOLO: conf, iou, imgsz; U‑Net: threshold), устройство, морфологию, режим отрисовки. Пресеты хранятся в `presets_deep_learning.json`.

11. **Гистограмма** – отображает распределение яркостей исходного изображения.

12. **Лог** – стандартный, с подробными сообщениями о загрузке модели, предсказании, количестве найденных объектов.

### Рабочий процесс

1. Загрузить изображения (папку или файлы).
2. Выбрать тип модели (U‑Net, YOLO-seg, DeepLabV3+, Custom ONNX).
3. Указать путь к файлу модели (для U‑Net, YOLO, ONNX). Для DeepLabV3+ путь не нужен.
4. Настроить параметры инференса (YOLO: пороги, размер входа; U‑Net: порог бинаризации).
5. Выбрать устройство (Auto обычно оптимально).
6. Нажать **Segment Current Image** – через несколько секунд появится маска.
7. Применить морфологию, инверсию, выбрать режим отрисовки объектов.
8. При необходимости сохранить аннотации.

### Особенности реализации

- **U‑Net**:
  - Используется библиотека `segmentation_models_pytorch`. Модель создаётся с параметрами из метафайла, затем веса загружаются из `.pth`. Энкодер и размер входа в UI заблокированы, чтобы избежать несоответствия с обученной моделью.
  - Если маска вероятности (бинарная) – применяется порог; если многоклассовая – выбирается класс с максимальной вероятностью (`argmax`).
- **YOLO-seg**:
  - Используется `ultralytics.YOLO`. Поддерживаются любые версии YOLO (8, 11, 26 и т.д.) с сегментационными возможностями.
  - Предсказание возвращает маски для каждого объекта; они объединяются (`np.max` по первому измерению) и ресайзятся к исходному размеру.
- **DeepLabV3+**:
  - Загружается из `torchvision.models.segmentation` с предобученными весами ImageNet. Выходной сигнал – сигмоида (бинарная сегментация) или softmax – в зависимости от задачи.
- **Custom ONNX**:
  - Инференс через ONNX Runtime. Ожидается, что модель принимает вход `[1,3,H,W]` (нормализованный RGB) и возвращает одну маску (или вероятности) того же размера, которая затем ресайзится.
- **SegFormer и SAM** – заглушки; при выборе выводится сообщение о нереализованности.

### Зависимости

- Для U‑Net: `segmentation_models_pytorch`, `torch`, `torchvision`.
- Для YOLO-seg: `ultralytics`.
- Для ONNX: `onnxruntime`.
- Для всех PyTorch‑моделей требуется установленный PyTorch с соответствующим бэкендом (CUDA, MPS или CPU).

### Связь с общими инструментами

- Навигация, загрузка, лог, `SmartGraphicsView`, пресеты, сохранение аннотаций – см. `tools.md`.
- Функции `apply_morphology`, `segment_contours`, `segment_projections`, `segment_min_area_rect` – общие утилиты.
- Гистограмма, отображение координат, управление списком объектов – стандартные.
