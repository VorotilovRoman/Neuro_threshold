
# deep_learning_tools.md

## Инструменты для работы с моделями глубокого обучения

Данный документ описывает модули и классы, обеспечивающие загрузку моделей, выполнение инференса, обучение (в отдельных виджетах) и настройку параметров.

### 1. Сегментаторы (`deep_learning_ops.py`)

#### Базовые классы
- `BaseSegmentor` – абстрактный класс с методами `load()` и `predict()`. Все сегментаторы наследуются от него.
- `TorchSegmentor` – база для PyTorch‑моделей. Содержит методы `_get_device()` (поддержка "auto", CUDA, MPS, CPU) и `_preprocess()` (преобразование BGR→RGB, ресайз, нормализация ImageNet).

#### Конкретные сегментаторы
- **`UNetSegmentor`**:
  - `load(weights_path)` – обязателен метафайл `*_meta.json` (содержит `encoder`, `input_size`, `num_classes`, опционально `decoder_channels`).
  - `predict(image, threshold=0.5)` – возвращает бинарную маску (0/255). Для многоклассовой сегментации применяет `argmax` и масштабирует индексы.
- **`DeepLabV3Segmentor`**:
  - `load(backbone='resnet50', output_stride=16, input_size=520)` – загружает предобученную модель из `torchvision`.
  - `predict(image)` – возвращает маску (сигмоида → ресайз).
- **`YOLOSegmentor`**:
  - `load(model_path)` – загружает модель через `ultralytics.YOLO`.
  - `predict(image, conf=0.25, iou=0.45, imgsz=640, save=False)` – возвращает объединённую маску всех обнаруженных объектов.
- **`ONNXSegmentor`**:
  - `load(model_path)` – создаёт сессию ONNX Runtime.
  - `predict(image)` – выполняет инференс, ресайзит результат к исходному размеру.

**Фабрика:** `create_segmentor(model_type, **kwargs)` – возвращает экземпляр сегментатора по строковому имени.

### 2. Виджеты настроек инференса (`DL_predict_settings.py`)

Все виджеты наследуются от `BaseInferenceSettings` (QGroupBox) и содержат:
- Поле выбора файла модели (QLineEdit + кнопка «Обзор»).
- Дополнительные параметры (специфичные для модели).
- Сигналы об изменении параметров (`model_path_changed`, `conf_changed` и т.д.).
- Метод `get_params()` – возвращает словарь с текущими настройками.

**Реализованные виджеты:**
- `YOLOInferenceSettings`: conf, iou, imgsz, save.
- `UNetInferenceSettings`: порог threshold (добавлен специально для бинаризации).
- `DeepLabV3InferenceSettings`: backbone, output stride (только информационные, заблокированы).
- `SegFormerInferenceSettings` – заглушка.
- `SAMInferenceSettings` – заглушка.
- `CustomONNXInferenceSettings` – только путь к файлу.

В основной вкладке (`DeepLearningWindow`) эти виджеты подключаются и их параметры передаются в сегментатор.

### 3. Виджеты обучения (`DL_train_settings_base.py` и `DL_train_settings_new_model.py`)

Эти виджеты **не используются** в текущей вкладке «Deep Learning» (она предназначена только для инференса), но они являются частью проекта и могут быть задействованы на вкладке «Обучение модели». Для полноты описания:

#### `BaseTrainWidget` (абстрактный)
Предоставляет общие элементы:
- Поля: папка проекта, имя эксперимента, устройство, эпохи, batch size (с чекбоксом Auto), кнопки «Запустить обучение», «Остановить», «Открыть папку проекта», прогресс‑бар.
- Сигналы: `run_training(params)`, `stop_training()`.
- Метод `set_running(running)` – управление состоянием кнопок.

#### Конкретные виджеты (в `DL_train_settings_base.py`):
- **`YOLOTrainWidget`** – обучение YOLO (любой версии). Настройки: датасет (папка или YAML), предустановленная модель или свой `.pt`, параметры аугментации (mosaic, close_mosaic, disable_aug), lr0, lrf, optimizer.
- **`UNetTrainWidget`** – обучение U‑Net. Настройки: датасет (папка с images/ и masks/), энкодер, размер входа, число классов, функция потерь, dropout, lr, optimizer, аугментации (hflip, vflip, rotate, scale, noise).

#### Виджеты в `DL_train_settings_new_model.py`:
- **`DeepLabV3TrainWidget`** – обучение DeepLabV3+. Расширенные настройки: backbone, output stride, output activation (argmax/sigmoid/softmax), веса классов (auto/from file), двухфазное обучение (заморозка backbone), комбинированные функции потерь (CrossEntropy+Dice+Focal с весами), планировщик (Poly, Step, Cosine).
- **`SegFormerTrainWidget`** – заглушка (требует реализации).
- **`SAMTrainWidget`** – заглушка.

Все виджеты обучения собирают параметры в словарь, который затем используется в рабочих потоках (не входят в представленные файлы, но предполагаются).

### 4. Рабочие потоки (не представлены в коде, но упомянуты)

В проекте существуют (или предполагаются) отдельные классы для фонового обучения:
- `YOLOTrainingWorker`, `UNetTrainingWorker`, `DeepLabV3TrainingWorker` – запускаются в отдельном потоке через `QThread`, получают словарь параметров и возвращают сигналы прогресса, завершения, ошибок.
- Они используют те же библиотеки (`ultralytics`, `segmentation_models_pytorch`, `torchvision`).

### 5. Взаимодействие с основной вкладкой

- `DeepLearningWindow` (см. `deep_learning_methods.py`) использует виджеты настроек инференса, но **не использует** виджеты обучения – для обучения существует отдельная вкладка «Обучение модели» (в `main.py` – `yolo_train`).
- При выборе модели в выпадающем списке соответствующий виджет настроек становится видимым, остальные скрываются.
- При изменении пути к модели (через «Обзор») вызывается `load_model_from_widget()`, которая создаёт сегментатор через фабрику, загружает веса и сохраняет объект в `self.segmentor`.
- При нажатии «Segment Current Image» вызывается `run_segmentation()`, которая передаёт параметры из виджета в метод `predict()` сегментатора.

### 6. Примечания по совместимости

- Для работы U‑Net необходима установка `segmentation_models_pytorch` и создание метафайла при обучении (автоматически или вручную).
- YOLO‑модели могут быть как официальными (загружаются автоматически по имени, например `yolov8n-seg.pt`), так и пользовательскими (указывается полный путь).
