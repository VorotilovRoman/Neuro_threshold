# demo_ensemble_methods.md

## Вкладка «Демонстрация результата»

Вкладка предназначена для визуального сравнения и объединения (ансамблирования) результатов детекции объектов, полученных от **YOLO** (модель детекции) и **классических пороговых методов** (пресетов из вкладки «Пороговые методы»). Пользователь загружает изображения (с возможностью автоматического показа истинных аннотаций), запускает инференс YOLO, применяет выбранный пресет (набор параметров бинаризации и морфологии) и затем комбинирует детекции обоих источников с помощью Weighted Boxes Fusion (WBF) или Non‑Maximum Suppression (NMS). Результаты отображаются в четырёх окнах, а координаты детекций выводятся в текстовых полях. Все настройки (веса, пороги, метод ансамблирования) настраиваются в реальном времени.

### Основные возможности

#### 1. Загрузка изображений и отображение истинных аннотаций
- Кнопки **Load Images** (выбор файлов) и **Load Folder** (выбор папки) – стандартные (см. `tools.md`).
- При загрузке изображений автоматически читаются истинные аннотации из `.txt`‑файлов, расположенных рядом с изображениями (YOLO‑формат, поддерживаются типы `detect`, `obb`, `segment`).
- Левое верхнее окно **«Исходное + истинные координаты»** отображает:
  - Само изображение.
  - Истинные аннотации (зелёные прямоугольники/полигоны).
  - Ниже – текстовое поле со списком истинных аннотаций в YOLO‑формате (для `obb` и `segment` – сокращённо).

#### 2. Настройка модели YOLO (детекция)
- **Модель YOLO** – виджет `YOLOInferenceSettings`:
  - Выбор файла `.pt` (кнопка «Обзор»).
  - Порог уверенности (0.01–1.0).
  - Порог IoU для NMS (0.01–1.0).
  - Размер входа (32–1280).
  - Опция «Сохранять результаты» (сохраняет предсказания YOLO на диск).
- **Device** – выбор устройства выполнения: Auto, CPU, CUDA, MPS.
- **Запустить YOLO** – выполняет инференс на текущем изображении. Результат отображается в окне **«YOLO инференс + координаты из модели»** (изображение с нарисованными боксами) и в текстовом поле (координаты в YOLO‑формате с уверенностью).

#### 3. Применение пресета пороговых методов
- **Пресеты обработки** – выпадающий список, содержащий все пресеты, сохранённые в глобальном менеджере пресетов (из вкладки «Пороговые методы»).
- При выборе пресета автоматически применяется соответствующая бинаризация, морфология и выделение объектов (в режиме `draw=False`). Результат:
  - Окно **«Пресет + координаты по пресету»** – изображение, где фон заменён на белый, а найденные объекты выделены (цвет из глобальных настроек).
  - Текстовое поле – список детекций в формате YOLO (тип `detect`, так как пресет возвращает прямоугольники). Для OBB/полигонов они преобразуются в bounding box.
- Пресет может включать любые параметры: метод бинаризации, инверсию, close/open факторы, режим отрисовки (используется для получения объектов, но итоговый вывод всегда прямоугольники).

#### 4. Ансамбль детекций (YOLO + пресет)
- **Ансамбль детекций** – объединяет детекции YOLO и все пресеты, для которых задан ненулевой вес.
- Настройки ансамбля:
  - **Вес YOLO** – коэффициент вклада модели (0.0–2.0).
  - Для каждого пресета из списка (динамически подгружаются) можно задать:
    - **Вес** (0.0–1.0) – вклад в финальное объединение.
    - **Уверенность** (0.0–1.0) – порог отбора детекций пресета (объекты с уверенностью ниже отбрасываются).
  - **Нормализовать** – приводит сумму весов всех источников к 1.0.
  - **Порог уверенности** – итоговый порог для финальных детекций (после объединения).
  - **Порог IoU (NMS)** – порог пересечения для группировки боксов.
  - **Учитывать вложенность** – для WBF дополнительно объединять боксы, если центр одного лежит внутри другого.
  - **Метод объединения**:
    - `WBF (Weighted Boxes Fusion)` – взвешенное усреднение координат и уверенностей.
    - `NMS (Non-Maximum Suppression)` – классическое подавление немаксимумов (без усреднения).
- **Запустить ансамбль** – выполняет:
  1. Запуск YOLO инференса (если ещё не был выполнен для текущего изображения).
  2. Применение всех выбранных пресетов.
  3. Объединение детекций согласно настройкам.
  4. Отображение результата в окне **«Ансамбль + координаты по ансамблю»** (изображение с нарисованными итоговыми боксами) и текстовое поле со списком детекций.

#### 5. Управление отображением
- **Reset zoom in all views** – сбрасывает масштаб во всех четырёх графических окнах.
- В каждом окне есть кнопка **▼** (слева в заголовке) – скрывает/показывает текстовое поле с координатами.
- Лог выполнения (правая панель, внизу) – выводит сообщения о загрузке модели, инференсе, применении пресетов, ошибках. Можно скрыть/показать кнопкой «▼ Лог выполнения».

#### 6. Автоматический запуск YOLO при смене изображения
- При переключении на другое изображение (кнопки Previous/Next, ввод номера страницы) YOLO инференс запускается **автоматически**, если выбрана модель и изображение загружено. Это позволяет быстро просматривать результаты по всей папке без лишних кликов.
- Принудительный запуск кнопкой «Запустить YOLO» также доступен.

### Рабочий процесс (типичный)

1. Загрузить папку с изображениями (и, желательно, с истинными аннотациями `.txt`).
2. Выбрать модель YOLO (`.pt`) и при необходимости настроить устройство.
3. Выбрать активный пресет (по умолчанию – «default»).
4. (Опционально) Настроить веса YOLO и пресетов, пороги, метод ансамблирования.
5. Нажать **Запустить ансамбль** – система выполнит YOLO, применит пресет, объединит результаты и покажет итог.
6. Переключаться между изображениями – YOLO будет перезапускаться автоматически, ансамбль можно запустить заново (или использовать уже готовый результат YOLO).
7. Анализировать качество: сравнить истинные аннотации (окно 1), предсказания YOLO (окно 2), пресета (окно 3) и ансамбля (окно 4).

### Особенности реализации

- **Типы аннотаций** – вкладка работает только с детекцией (`detect`). Пресеты, возвращающие `obb` или `segment`, преобразуются в охватывающие прямоугольники.
- **Автоматическое обновление пресетов** – при переключении на вкладку или получении фокуса список пресетов перезагружается из глобального менеджера, что позволяет использовать пресеты, добавленные в других вкладках.
- **Потоки** – YOLO инференс и ансамбль выполняются в отдельных `QThread`, чтобы не блокировать интерфейс. Прогресс отображается в прогресс‑баре.
- **Остановка операций** – если инференс или ансамбль уже запущены, повторный запуск останавливает предыдущий поток.
- **Истинные аннотации** – при загрузке изображений автоматически считываются из `.txt`‑файлов. Для OBB и полигонов отображается сокращённое текстовое представление.

### Зависимости

- `ultralytics` – для загрузки и инференса YOLO.
- PyTorch, OpenCV, NumPy – стандартные.
- `ensemble_utils.py` – функции `get_detections_from_preset`, `ensemble_detections`, `weighted_boxes_fusion`, `non_max_suppression`.

### Примечания

- Вкладка предназначена **только для детекции** (bounding boxes), а не для сегментации. Маски, возвращаемые YOLO‑seg, не используются (модель должна быть детекционной).
- Если в пресете выбран режим отрисовки `None`, он не будет генерировать объекты.
- Для нормальной работы пресетов необходимо, чтобы в глобальном менеджере пресетов (`PresetManager`) были сохранены хотя бы один пресет. Иначе раздел весов пресетов будет пуст.
- При изменении режима ресайза изображений (чекбокс «Resize to max 1024px» в навигации) программа предложит перезагрузить изображения (с потерей текущих результатов). Рекомендуется установить режим ресайза до загрузки изображений.

### Связь с общими инструментами

- Навигация, загрузка, лог, `SmartGraphicsView` – см. `tools.md`.
- Пресеты – используются из `PresetManager`.
- Функции `apply_threshold_method`, `apply_morphology`, `segment_contours`, `segment_projections`, `segment_min_area_rect` – общие утилиты.
- `ensemble_utils` – специальные функции для ансамблирования.
