# gradient_methods.md

## Вкладка «Градиентные методы»

Вкладка предназначена для сегментации изображений с помощью выделения границ (edges), различных дифференциальных операторов, последующей бинаризации, морфологической обработки, фильтрации объектов и сохранения аннотаций в формате YOLO. Она позволяет гибко настраивать параметры каждого метода, визуализировать результаты в шести окнах и автоматически перебирать комбинации параметров.

### Основные возможности

1. **Загрузка изображений**  
   - Кнопки «Load Images» и «Load Folder» (стандартные, см. `tools.md`).  
   - Опция «Resize to max 1024px» – уменьшает большие изображения для ускорения.

2. **Выбор градиентного оператора**  
   Выпадающий список содержит следующие методы:
   - **Sobel** – вычисление градиента с помощью производных Собеля.
   - **Scharr** – улучшенный оператор Собеля (более точный).
   - **Laplacian** – оператор Лапласа (вторая производная).
   - **Canny** – многоэтапный детектор границ (шумоподавление, немаксимумы, гистерезис).
   - **Prewitt** – оператор Превитта (аппроксимация градиента).
   - **Roberts** – оператор Робертса (кросс-разности).
   - **Kirsch** – нелинейный оператор на основе 8 направлений.

3. **Параметры методов** (отображаются только для выбранного оператора)

   - **Sobel**:
     - Kernel size (нечётное, 1…31) – размер ядра Собеля.
     - Scale (1…10) – масштабирующий множитель результата.
   - **Scharr**:
     - Scale (1…10) – аналогично.
   - **Laplacian**:
     - Kernel size (нечётное, 1…31).
     - Scale (1…10).
   - **Canny**:
     - Threshold1 (0…255) – нижний порог гистерезиса.
     - Threshold2 (0…255) – верхний порог.
     - Aperture size (3,5,7) – размер апертуры для производных.
   - **Prewitt**, **Roberts**, **Kirsch** – не имеют настраиваемых параметров.

4. **Бинаризация градиентной маски**  
   - Полученное изображение градиента (интенсивности) преобразуется в бинарное с помощью **порога** (Gradient threshold). Слайдер 0…255.

5. **Морфологическая обработка**  
   Применяется к бинарному изображению (после порога) для улучшения связности контуров:
   - **Closing factor** – относительный размер ядра закрытия (0…0.05 от минимальной стороны изображения). Шаг 0.005.
   - **Opening factor** – относительный размер ядра открытия (0…0.05).
   - **Kernel shape** – форма ядра: Rectangle, Ellipse, Cross.

6. **Фильтрация объектов по площади**  
   - **Min contour area (% of image)** – удаляет контуры, площадь которых меньше указанного процента от общей площади изображения. Диапазон 0.0–0.1%, шаг 0.002%. Значение 0 отключает фильтрацию.

7. **Заливка контуров**  
   - **Fill closed contours** – если включено, то все замкнутые контуры заливаются сплошным белым цветом (создаются маски объектов). После заливки может быть применено закрытие для сглаживания.

8. **Инверсия маски**  
   - **Invert mask** – инвертирует итоговую бинарную маску (чёрное ↔ белое).

9. **Режим отрисовки объектов**  
   Определяет, как объекты (найденные на основе итоговой маски) будут визуализированы и сохранены:
   - **None** – объекты не выделяются.
   - **Segmentation (Polygon)** – контуры объектов (полигоны) – тип `segment` в YOLO.
   - **Bounding Box (Detect)** – ограничивающие прямоугольники – тип `detect`.
   - **OBB (Oriented Box)** – ориентированные прямоугольники (минимальный повёрнутый прямоугольник) – тип `obb`.
   - **Use convex hull** – для режимов Polygon и OBB строить выпуклую оболочку контура.

10. **Визуализация (6 окон)**  
    В левой части расположена сетка 3×2:
    - **Оригинал** – исходное цветное изображение.
    - **Градиент** – результат применения выбранного оператора (интенсивность градиента).
    - **Бинарное** – результат пороговой обработки градиента.
    - **Морфология** – результат после closing/opening.
    - **Заливка** – результат после удаления мелких контуров, заливки и инверсии (итоговая маска объектов).
    - **Аннотированное** – исходное изображение с нарисованными объектами (цвет, толщина – из глобальных настроек).  
    Каждое окно поддерживает:
      - Зум: колёсико мыши.
      - Панорамирование: перетаскивание левой кнопкой.
      - Сброс зума для всех окон: кнопка «Reset zoom in all views».

11. **Список объектов и координаты**  
    - Справа отображается список найденных объектов с чекбоксами. Выбранные объекты подсвечиваются на аннотированном изображении.
    - В текстовом поле ниже выводятся координаты выбранных объектов (для `detect` – пиксельные прямоугольники, для `obb`/`segment` – нормализованные точки).

12. **Сохранение аннотаций**  
    Кнопка «Save Labels» сохраняет текущие объекты в YOLO‑формате в файл `.txt` рядом с изображением.

13. **Пресеты**  
    - Выпадающий список пресетов (сохраняются все настройки: метод, параметры, порог, морфология, фильтрация, заливка, инверсия, режим отрисовки, hull).
    - «Сохранить как пресет», «Удалить пресет».
    - Пресеты хранятся в файле `presets_gradient.json`.

14. **Автоподбор параметров (Auto Research)**  
    - Включение галочкой «Enable auto research».
    - Кнопка «Configure auto research» – открывает диалог, где можно для каждого градиентного метода, бинаризации, фильтрации объектов, морфологии и инверсии задать:
      - включён/выключен;
      - диапазоны параметров (начало, конец, шаг);
      - для `fill_contours` и `kernel_shapes` – перебор всех возможных значений.
    - Кнопка «Запустить автоподбор» – выполняет перебор всех комбинаций для текущего изображения или всех загруженных (опция «Search all images»).
    - Результат сохраняется в файл `_gradient_report.txt` в папке изображения, где для каждой комбинации записано количество найденных объектов и их координаты.

15. **Гистограмма**  
    Показывается распределение яркостей исходного изображения (не градиента). Можно скрыть/показать кнопкой «Показать гистограмму».

16. **Лог выполнения**  
    Текстовое окно с сообщениями. Скрывается/показывается кнопкой «Показать лог».

### Рабочий процесс (кратко)

1. Загрузить изображения (папка или файлы).
2. Выбрать градиентный метод, настроить его параметры.
3. Установить порог бинаризации.
4. При необходимости применить морфологию (закрытие/открытие).
5. Включить/выключить заливку контуров и фильтрацию по площади.
6. Выбрать режим отрисовки объектов (полигоны, прямоугольники или OBB).
7. Просмотреть результаты в 6 окнах.
8. При необходимости сохранить пресет или запустить автоподбор.
9. Сохранить аннотации в YOLO‑формате.

### Особенности реализации

- Все градиентные операторы (кроме Canny) возвращают **магнитуду градиента** (значения 0–255), которая затем бинаризуется. Для Canny сразу получается бинарная карта границ.
- Перед вычислением градиента изображение преобразуется в оттенки серого и инвертируется (`cv2.bitwise_not`), чтобы тёмные объекты на светлом фоне обрабатывались корректно (это особенность данной вкладки).
- Морфологические операции применяются **после бинаризации**, но до заливки контуров.
- Заливка контуров (`Fill closed contours`) сначала находит все внешние контуры, затем закрашивает их области белым цветом. После заливки опционально выполняется закрытие для удаления мелких отверстий.
- Удаление мелких контуров работает **после морфологии** (но до заливки) и использует процент от общей площади изображения.
- Итоговая маска объектов – результат после заливки и инверсии.
- Аннотации сохраняются в формате YOLO с классом 0 (по умолчанию). Пользователь может вручную отредактировать файл или использовать другие вкладки для присвоения классов.

### Связь с общими инструментами

- Навигация, загрузка, лог, `SmartGraphicsView` – см. `tools.md`.
- Сохранение/загрузка аннотаций, пресеты, автоподбор реализованы аналогично вкладке «Пороговые методы», но с адаптацией под градиенты.
- Глобальные настройки (цвета, толщина линий) влияют на отображение аннотированного окна.
