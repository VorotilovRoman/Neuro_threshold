# interactive_methods.md

## Вкладка «Интерактивные методы»

Вкладка предназначена для сегментации изображений с помощью интерактивных алгоритмов, требующих от пользователя задания начальных меток (прямоугольник, штрихи переднего/фонового плана). После выполнения сегментации результат может быть обработан морфологией, отрисован в виде объектов (контуры, прямоугольники, ориентированные боксы) и сохранён в YOLO‑формате.

### Основные возможности

1. **Загрузка изображений** – стандартная (кнопки «Load Images», «Load Folder», см. `tools.md`), с опциональным ресайзом до 1024 px.

2. **Выбор метода сегментации** (выпадающий список):
   - **GrabCut** – итеративный графовый разрез с учётом цветовых моделей.
   - **Lazy Snapping** – графовый разрез на суперпикселях (требует штрихов FG/BG).
   - **SuperCut** – вариант Lazy Snapping с дополнительными параметрами (требует прямоугольника).
   - **OneCut** – сегментация на основе GMM цветовых моделей фона/объекта (требует прямоугольника).
   - **Random Walker** – случайный блуждающий по графу (требует штрихов FG/BG).
   - **Watershed (marker)** – водораздел на основе расстояния до фона (автоматический, без ввода).
   - **Active Contours** – активные контуры (змейки), инициализируемые прямоугольником.

3. **Интерактивный ввод** – задаётся через три режима (кнопки с проверкой состояния):
   - **Draw Bounding Box** – рисование прямоугольника (левой кнопкой, один раз, после чего инструмент автоматически сбрасывается).
   - **Scribble FG** – нанесение зелёных штрихов переднего плана (повторное нажатие выключает режим).
   - **Scribble BG** – нанесение синих штрихов фона.
   - **Pan/Zoom** – режим панорамирования и масштабирования (по умолчанию).
   
   Требования к вводу динамически отображаются в виджете «Требования», а кнопки окрашиваются:
   - зелёным – если необходимый ввод присутствует;
   - красным – если обязательный ввод отсутствует;
   - серым – если ввод не требуется для выбранного метода.
   
   Кнопка **Run Segmentation** активна только при выполнении требований.

4. **Параметры методов** – сгруппированы в панели, отображаемые по мере необходимости:
   - **GrabCut**: итерации (1–20), режим (`GC_INIT_WITH_RECT`, `GC_INIT_WITH_MASK`, `Both`).
   - **Lazy Snapping / SuperCut / OneCut**: размер суперпикселя (10–200), компактность (1–50), сглаживание sigma (0–5), цветовая sigma (0.1–50).
   - **SuperCut**: дополнительно lambda (0–10) – вес унарного члена, sigma цвета (0–100).
   - **OneCut**: пространственный вес (0–10) и вес данных (0–10).
   - **Watershed**: порог расстояния (1–100% от максимума), минимальная площадь маркера (1–100).
   - **Random Walker**: beta (0–1000) – вес рёбер, режим решателя (`cg_j`, `cg_mg`, `bf`).
   - **Active Contours**: alpha (эластичность, 0–1), beta (жёсткость, 0–1), gamma (шаг, 0–1), max_iter (50–1000), порог сходимости (0.0001–0.1), тип начального контура (Rectangle, Circle, Ellipse), опция сглаживания изображения.

5. **Визуализация (4 окна)** – в левой части:
   - **Оригинал** – исходное изображение с нарисованными поверх него прямоугольником и штрихами (временные штрихи отображаются жёлтым/голубым).
   - **Сегментация** – бинарная маска, полученная алгоритмом (без постобработки).
   - **Морфология** – маска после закрытия/открытия и инверсии.
   - **Аннотированное** – исходное изображение, на котором белым цветом выделены области объектов (фон заменён на белый) и отрисованы выбранные объекты (контуры, прямоугольники, OBB).
   
   Все окна поддерживают зум/панорамирование, сброс зума общей кнопкой.

6. **Постобработка маски**:
   - **Closing factor** (0–0.10) – относительный размер ядра закрытия.
   - **Opening factor** – относительный размер ядра открытия.
   - **Kernel shape** – Rectangle, Ellipse, Cross.
   - **Invert mask** – инверсия итоговой бинарной маски (после морфологии).

7. **Выделение и отрисовка объектов**:
   - **Draw objects** – режим отрисовки на аннотированном изображении:
     - `None` – объекты не рисуются.
     - `Contours (simple)` – полигоны контуров (тип `segment`).
     - `Projections (connected components)` – ограничивающие прямоугольники (тип `detect`).
     - `Min area rectangle` – ориентированные прямоугольники (тип `obb`).
   - **Use convex hull** – для контуров и OBB строить выпуклую оболочку.
   - Список объектов (с чекбоксами) отображается справа, выбранные объекты подсвечиваются.
   - Координаты выбранных объектов выводятся в текстовое поле (для `detect` – пиксельные координаты, для `obb`/`segment` – нормализованные).

8. **Сохранение аннотаций** – кнопка «Save Labels» сохраняет текущий список объектов в YOLO‑формат (`.txt` рядом с изображением). Поддерживаются все три типа (`detect`, `obb`, `segment`).

9. **Пресеты** – сохранение и загрузка полной конфигурации (метод, параметры, морфология, режим отрисовки, hull). Пресеты хранятся в `presets_interactive.json`.

10. **Лог и гистограмма** – стандартные (скрываются по кнопкам). Гистограмма отображает распределение яркостей исходного изображения (в оттенках серого).

### Рабочий процесс (типичный)

1. Загрузить изображение (папку).
2. Выбрать метод сегментации.
3. Ввести необходимые метки: прямоугольник и/или штрихи FG/BG. Кнопки автоматически подсветятся.
4. При необходимости настроить параметры метода.
5. Нажать **Run Segmentation** – вычисляется маска.
6. Применить морфологию/инверсию, выбрать режим отрисовки объектов.
7. Выбрать объекты в списке (все по умолчанию выбраны) для просмотра координат.
8. Сохранить аннотации или продолжить работу с другими изображениями.

### Особенности реализации

- Все интерактивные операции (рисование, отображение временных фигур) выполняются через `SmartGraphicsView` (см. `tools.md`). Коллбэки обновляют внутренние массивы `rect`, `fg_scribbles`, `bg_scribbles` и вызывают обновление отображения.
- После смены изображения или нажатия **Reset Input** все вводные данные (прямоугольник, штрихи, маска) сбрасываются.
- Для методов, требующих прямоугольника (`SuperCut`, `OneCut`, `Active Contours`), маска инициализируется этим прямоугольником; для `GrabCut` в режиме `GC_INIT_WITH_RECT` также используется прямоугольник.
- Метод `Watershed (marker)` не требует ввода – сегментация выполняется автоматически на основе преобразования расстояния и маркеров локальных максимумов.
- При переключении между изображениями маска и вводные данные сбрасываются.
- При изменении режима ресайза изображения предлагается перезагрузить (с потерей всех вводных данных).

### Зависимости

Для работы `Lazy Snapping`, `SuperCut`, `OneCut` требуется библиотека **networkx**. Если она не установлена, соответствующие методы будут недоступны (выводится предупреждение).

### Связь с общими инструментами

- Навигация, загрузка, лог, `SmartGraphicsView`, пресеты, сохранение аннотаций – см. `tools.md`.
- Глобальные настройки цвета и толщины линий (`settings`) влияют на отрисовку объектов в аннотированном окне.
- Функции `apply_morphology`, `segment_contours`, `segment_projections`, `segment_min_area_rect` используются для постобработки и выделения объектов (общие утилиты).
