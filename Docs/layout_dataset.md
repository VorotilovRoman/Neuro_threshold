# layout_dataset.md

## Вкладка «Добавить аннотации» (Image Labeler)

Вкладка предназначена для ручной разметки изображений в формате YOLO (detect, obb, segment). Пользователь загружает изображения (папку, файлы или датасет из YAML), рисует прямоугольные ограничивающие рамки (`detect`), редактирует существующие аннотации (перемещение, изменение размера, поворот для OBB, изменение точек полигона), удаляет объекты и сохраняет аннотации в текстовые файлы `.txt` рядом с изображениями. Поддерживаются три типа аннотаций, унифицированных в коде.

### Основные возможности

1. **Загрузка изображений**
   - **Load Images** – выбор отдельных файлов (стандартный диалог).
   - **Load Folder** – загрузка всех изображений из выбранной папки.
   - **Load YAML** – загрузка датасета по YAML‑файлу (формат YOLO, секции `train`, `val`, `test`). Изображения загружаются с учётом опции ресайза.
   - **Load Labels** – загрузка существующих аннотаций из выбранной папки (файлы `.txt` с именами, совпадающими с изображениями).

2. **Навигация по изображениям** – стандартный виджет `ImageNavigationWidget` (кнопки Previous/Next, поле номера страницы, опция ресайза). При переключении изображения текущие аннотации загружаются из памяти (`self.all_annotations`).

3. **Режимы работы**
   - **Drawing Mode** (кнопка «Drawing Mode») – включение рисования прямоугольников (`detect`). После нажатия курсор меняется на крестик, можно левой кнопкой мыши нарисовать прямоугольник (от точки до точки). После отпускания кнопки прямоугольник добавляется в список аннотаций, а режим рисования остаётся активным.
   - **Панорамирование и зум** – доступны всегда, когда режим рисования выключен. Перетаскивание левой кнопкой, зум колёсиком мыши. Сброс зума – кнопка «Reset zoom» (общая для всех вкладок).
   - **Редактирование аннотаций** – включается автоматически при выборе объекта (клик по нему на изображении или выбор в списке). Редактирование поддерживается для всех трёх типов (`detect`, `obb`, `segment`):
     - **detect**: можно перемещать весь прямоугольник, а также перетаскивать любой из четырёх углов (изменение размера).
     - **obb**: можно перемещать, изменять размеры за углы, вращать за специальную дугу (сектор) вокруг центра, а также растягивать за середины сторон.
     - **segment**: можно перемещать весь полигон, а также перетаскивать отдельные вершины.
   - Для выхода из режима редактирования – нажать Esc, правую кнопку мыши или выбрать другой объект (снятие выделения).

4. **Управление объектами**
   - Список объектов отображается справа (в группе «Objects»). Каждый элемент показывает тип, класс, координаты (в пикселях для detect, сокращённо для obb/segment).
   - **Выбор объекта** – клик по элементу списка подсвечивает соответствующий объект на изображении и включает режим редактирования.
   - **Удаление выбранного объекта** – кнопка «Delete Selected».
   - **Контекстное меню** – правый клик по объекту в списке → «Delete».
   - **Двойной клик по объекту** – открывает диалог смены class ID (для всех типов аннотаций). Позволяет изменить целочисленную метку класса.

5. **Сохранение аннотаций**
   - Кнопка **Save Labels** – сохраняет текущие аннотации (все объекты текущего изображения) в файл `.txt` рядом с изображением. Используется универсальная функция `save_annotations`, поддерживающая `detect`, `obb`, `segment`.
   - Сохранение выполняется в нормализованных координатах (0..1) в формате YOLO:
     - `detect`: `class cx cy w h`
     - `obb`: `class x1 y1 x2 y2 x3 y4` (8 чисел)
     - `segment`: `class x1 y1 x2 y2 ...` (чётное количество координат)

6. **Вспомогательные возможности**
   - **Гистограмма** – отображает распределение яркостей текущего изображения (можно скрыть/показать).
   - **Лог** – текстовое окно с сообщениями о загрузке, сохранении, добавлении/удалении объектов.
   - **Глобальные настройки** – цвета линий аннотаций, выделенных объектов, толщина линий и размер шрифта (через `settings`). При изменении настроек отображение обновляется.

### Рабочий процесс (типичный)

1. Загрузить изображения (папка, файлы или YAML‑датасет).
2. Если уже есть аннотации, они автоматически загружаются из `.txt` рядом с изображениями (если загрузка через папку/файлы) или через «Load Labels».
3. Включить **Drawing Mode** (по умолчанию включен).
4. Нарисовать прямоугольники вокруг объектов (один или несколько).
5. При необходимости отредактировать объекты:
   - Выбрать объект (клик на изображении или в списке).
   - Перемещать/изменять/вращать (для OBB) с помощью мыши.
   - Двойной клик в списке – изменить class ID.
   - Удалить ненужные объекты (Delete Selected или контекстное меню).
6. Переключиться на следующее изображение (кнопки навигации) – аннотации предыдущего автоматически сохраняются в памяти.
7. По завершении работы нажать **Save Labels**, чтобы записать аннотации на диск (для текущего изображения). Если нужно сохранить все – следует сохранять каждое изображение при переключении или в конце.

### Особенности реализации

- **Унифицированные аннотации**: все объекты хранятся в виде кортежей:
  - `('detect', class_id, cx, cy, w, h)`
  - `('obb', class_id, [x1,y1,x2,y2,x3,y3,x4,y4])`
  - `('segment', class_id, [x1,y1,x2,y2,...])`
- Для отображения и редактирования используется `SmartGraphicsView` (см. `tools.md`). Этот виджет поддерживает:
  - Отрисовку аннотаций разных типов.
  - Интерактивное выделение и редактирование (перемещение, масштабирование, вращение OBB, перемещение вершин полигонов).
  - Коллбэки `on_rect_drawn`, `on_annotation_modified`, `on_selection_changed`, `on_reset_tool`.
- При добавлении прямоугольника (`detect`) координаты сразу нормализуются (деление на ширину/высоту изображения).
- При загрузке существующих аннотаций (из `.txt`) они парсятся функцией `load_annotations`, которая автоматически определяет тип по количеству чисел.
- При изменении режима ресайза (чекбокс «Resize to max 1024px») изображения перезагружаются, а все несохранённые аннотации теряются – перед этим выдаётся предупреждение.

### Управление клавиатурой

- **Esc** – выход из режима редактирования (снять выделение). Если активен режим рисования – выключить его (переход в панорамирование).
- **Ctrl + ЛКМ** – выделение объекта под курсором (если не в режиме рисования).
- **Правая кнопка мыши** – если активен режим рисования или редактирования, сбросить инструмент и перейти в панорамирование.

### Связь с общими инструментами

- Навигация, загрузка, лог, `SmartGraphicsView`, сохранение аннотаций – см. `tools.md`.
- Функции `load_annotations`, `save_annotations`, `load_images_universal`, `load_dataset_from_yaml` – из `utils.image_io`.
- Глобальные настройки цвета и толщины (`settings`) влияют на отображение аннотаций.
- Гистограмма и кнопки лога – стандартные для всех вкладок.

### Примечания

- Вкладка **не** поддерживает автоматическое выделение объектов (это функционал других вкладок). Здесь только ручная разметка.
- Тип создаваемых объектов – всегда `detect` (прямоугольник). Для создания OBB или полигонов следует использовать другие вкладки (например, «Пороговые методы» с режимом отрисовки OBB или полигонов, затем редактировать здесь). Однако текущая реализация позволяет **редактировать** OBB и полигоны, импортированные из файлов.
- При загрузке датасета через YAML пути к изображениям и аннотациям определяются согласно стандартной структуре YOLO (папки `images/train`, `labels/train` и т.д.). Маски не загружаются.
- Для работы с YAML‑файлом требуется наличие `path` и секций `train`/`val`/`test` в YAML.
