# model_research.md

## Вкладка «Анализ отчётов»

Вкладка предназначена для глубокого анализа результатов автоматического перебора параметров (`_full_report.txt`), полученных на вкладках «Пороговые методы» и «Градиентные методы». Позволяет загружать данные, фильтровать, вычислять метрики (F1, mAP), кластеризовать параметры, строить регрессионные модели для предсказания качества и отбирать оптимальные разнообразные пресеты для каждого метода.

### Основные возможности

1. **Загрузка данных**
   - **Выбрать папку** – сканирует выбранную папку на наличие файлов `*_full_report.txt` (созданных авто‑исследованием) и соответствующих файлов истинных аннотаций (`*.txt`). Парсит файлы, извлекает параметры методов, количество объектов и сохраняет всё в DataFrame.
   - **Загрузить датасет** – загружает ранее сохранённый датасет (CSV или Pickle).
   - **Сохранить датасет** – сохраняет текущий DataFrame (после всех фильтраций и обработок) в файл.

2. **Отображение выборок**
   - **Первые 100** / **Последние 100** / **Случайные 100** – показывают соответствующие строки в таблице.
   - **Показать всё** – отображает весь датасет (осторожно при большом количестве строк).
   - **Сохранить отображаемые** – сохраняет текущий отображаемый фрагмент (например, отфильтрованный) в файл.

3. **Фильтрация датасета** (кнопки слева)
   - **Удалить: num_objects=0 и true>=1** – строки, где предсказаний нет, но истинные объекты есть.
   - **Удалить: pred > 2*true** – сильная oversegmentation (предсказаний больше чем в 2 раза больше истинных).
   - **Удалить дубликаты строк** – полные дубликаты (по всем колонкам).
   - **Удалить некорректные параметры** – строки, где параметры метода не удаётся распарсить (например, нечисловые значения).
   - **Удалить: num>15 (объектов или истинных)** – аномально большое количество объектов (порог 15).
   - **Удалить: объект на весь снимок** – среди предсказаний есть объект с центром (0.5,0.5) и размерами (1,1) (покрывает всё изображение).
   - **Удалить строки без истинных объектов (num_true_objects == 0)** – изображения, где нет ни одной истинной аннотации.
   - **Случайная выборка: N строк** – оставить в датасете только N случайных строк (полезно для ускорения экспериментов).

   Все фильтры работают последовательно, обновляя таблицу и счётчик строк.

4. **Расчёт метрик (meanAP)**
   - Кнопка **Расчет meanAP** – для каждой строки вычисляет:
     - `f1_10`, `f1_20`, ..., `f1_90` – F1‑мера при порогах IoU 0.1…0.9.
     - `f1_50` – для совместимости (IoU=0.5).
     - `mean_f1` – среднее арифметическое F1 по всем девяти порогам.
     - `avg_iou_tp` – средний IoU по истинно‑положительным детекциям (при пороге 0.5).
   - После расчёта появляется возможность фильтровать строки по порогу `mean_f1` (слайдер + кнопка **Удалить строки с mean_f1 < порога**).

5. **Кластеризация параметров**
   - **Кластеризация параметров** – для каждого метода извлекает числовые параметры (например, `window`, `k`, `threshold`), масштабирует их и выполняет K‑Means (количество кластеров = min(10, max(2, число_строк//5)). Добавляет колонку `cluster`. Для каждого метода сохраняются:
     - Текстовый отчёт (`clusters/clusters_<method>_report.txt`) – диапазоны параметров, статистика F1.
     - График кластеров (1D или PCA‑проекция) в `clusters/clusters_<method>.png`.
   - **Кластеризация open/close** – для каждой группы (method, cluster) выполняет K‑Means по признакам `close_factor` и `open_factor` (количество кластеров 3). Добавляет колонку `cluster_co`.

6. **Агрегация по параметрам**
   - **Агрегировать по параметрам** – группирует строки по уникальным комбинациям `method, cluster, cluster_co, invert` (или по полным параметрам, если кластеры не вычислены) и оставляет в каждой группе **топ‑K** строк с наибольшим `mean_f1` (K задаётся в поле «Топ‑K для агрегации»). Позволяет сократить датасет до лучших представителей каждой кластерной группы.

7. **Оптимизация инверсии**
   - **Оптимизировать инверсию** – для каждого метода сравнивает средний `mean_f1` при `invert=False` и `invert=True`. Оставляет только строки с лучшим значением (если равны – оставляет более частый вариант). Удаляет все строки с худшей инверсией.

8. **Сохранение уникальных параметров**
   - **Сохранить уникальные params** – для каждого метода выписывает все уникальные строки `params` в текстовый файл (папка `unique_params/`). Полезно для анализа используемых комбинаций.

9. **Анализ модели (регрессия и топ‑пресеты)**
   - Использует колонки `cluster` и `cluster_co` как признаки (кодирует их one‑hot). Обучает **RandomForestRegressor** для предсказания `mean_f1` (или взвешенного F1) на основе комбинации кластеров.
   - Для каждого метода генерирует все возможные комбинации (`cluster`, `cluster_co`), предсказывает `pred_f1` и выбирает **топ‑5** комбинаций (с учётом разнообразия, если включена галочка).
   - Результат отображается в правом нижнем виджете: метод, ранг, параметры, инверсия, close/open факторы, реальный F1 (`actual_f1`), предсказанный F1 (`pred_f1`), кластеры.
   - **Галочки влияния**:
     - **Использовать разнообразие при выборе топ‑5** – жадный алгоритм, который выбирает комбинации, максимально отличающиеся друг от друга по методу, cluster и cluster_co.
     - **Штраф за низкое качество TP (mean_f1 * avg_iou_tp^0.5)** – вместо обычного `mean_f1` использует взвешенную метрику, штрафующую низкое качество истинно‑положительных детекций.
   - Кнопка **Запустить анализ модели** – выполняет описанный процесс и выводит топ‑5 для каждого метода.

10. **Глобальный топ‑10 (разнообразные пресеты)**
    - **Глобальный топ методов** – из загруженного датасета (уже содержащего колонки `actual_f1`, `cluster`, `cluster_co`, `method`) выбирает **10** строк, максимизируя разнообразие (расстояние между пресетами учитывает метод, cluster и cluster_co). Результат отображается в том же виджете.

11. **Загрузка топ‑пресетов по методам**
    - **Загрузить топ по методам** – загружает ранее сохранённый датасет с топ‑пресетами (например, результат предыдущего анализа). Этот датасет должен содержать колонки `method`, `rank`, `params`, `invert`, `close_factor`, `open_factor`, `actual_f1`, `pred_f1`, `cluster`, `cluster_co`. После загрузки становятся доступными:
      - **Удалить идеальные строки** – удалить строки, где `actual_f1` ≥ заданного порога (по умолчанию 1.0). Позволяет отбросить уже идеальные пресеты для дальнейшего улучшения.
      - **Глобальный топ методов** (работает и на этом датасете).

12. **Лог и таблица**
    - Вся информация о загрузке, фильтрации, расчётах выводится в лог (нижняя правая часть).
    - Основная таблица (верхняя) отображает текущий датасет (первые 100 или все строки). Колонки можно сортировать.

### Формат входных данных

- Файлы отчётов генерируются авто‑исследованием на вкладках «Пороговые методы» и «Градиентные методы». Имя файла: `<имя_изображения>_full_report.txt`.
- Каждый такой файл содержит блоки вида:
  ```
  === Метод: Simple Threshold, порог=127, инверсия=Нет, close=0.00, open=0.00 | Количество объектов: 5 ===
  # далее строки с координатами объектов в YOLO‑формате (class cx cy w h)
  ```
- Парсинг извлекает: метод, параметры, инверсию, close_factor, open_factor, количество предсказанных объектов и список объектов.
- Истинные аннотации загружаются из одноимённого `.txt`‑файла (рядом с отчётом).

### Рабочий процесс (типичный)

1. Выбрать папку с отчётами (`*_full_report.txt`) и аннотациями (`*.txt`) – нажать **Выбрать папку**. Данные автоматически загрузятся в таблицу.
2. (Опционально) Отфильтровать датасет (удалить строки с аномалиями, пустые изображения, oversegmentation и т.д.).
3. Нажать **Расчет meanAP** – добавить метрики.
4. (Опционально) Отфильтровать по `mean_f1` (низкое качество удалить).
5. Выполнить **Кластеризацию параметров** (добавится колонка `cluster`).
6. Выполнить **Кластеризацию open/close** (добавится колонка `cluster_co`).
7. (Опционально) **Агрегировать по параметрам** – оставить топ‑1 (или другой K) для каждой группы.
8. (Опционально) **Оптимизировать инверсию**.
9. Настроить галочки (разнообразие, штраф) и нажать **Запустить анализ модели**.
10. В виджете результатов увидеть топ‑5 пресетов для каждого метода.
11. **Сохранить топ‑пресеты** (кнопка «Сохранить топ‑пресеты» в виджете результатов) – сохранить результат в CSV/Pickle.
12. **Добавить в пресеты** – добавить выбранные комбинации в глобальный список пресетов (для использования на вкладках пороговых/градиентных методов).

### Зависимости

- Для регрессии: `scikit-learn` (RandomForestRegressor, StandardScaler, PCA, KMeans).
- Для кластеризации: `scikit-learn`.
- Для визуализации: `matplotlib` (сохраняет графики кластеров).

### Примечания

- Если данных для метода меньше 10 строк, анализ модели не обучает регрессию, а просто берёт топ‑5 по `mean_f1` (без разнообразия).
- Колонки `cluster` и `cluster_co` необходимы для работы регрессии. Если они не вычислены, анализ попытается использовать группировку по полным параметрам, но рекомендуется выполнить кластеризацию.
- При загрузке датасета из файла (не из папки) должны присутствовать колонки: `objects`, `true_objects`, `method`, `params`, `invert`, `close_factor`, `open_factor`. Остальные колонки могут быть добавлены позже.
- Для работы с топ‑пресетами (кнопка «Загрузить топ по методам») требуется специальный формат датасета, включающий `actual_f1`, `pred_f1`, `cluster`, `cluster_co`.
