# treshold_methods.md

## Вкладка «Пороговые методы»

Вкладка предназначена для бинаризации изображений (преобразования в чёрно-белый вид) с последующим выделением объектов, морфологической обработкой и сохранением аннотаций в формате YOLO. Она включает интерактивные средства настройки, визуализацию результатов в четырёх окнах, автоматический перебор параметров и управление пресетами.

### Основные возможности

1. **Загрузка изображений**  
   - Кнопки «Load Images» (выбор файлов) и «Load Folder» (выбор папки).  
   - Опция «Resize to max 1024px» – уменьшает большие изображения для ускорения обработки.

2. **Выбор метода бинаризации**  
   Выпадающий список содержит следующие методы (для каждого – свои параметры):
   - **Simple Threshold** – фиксированный порог (0…255).
   - **Otsu** – автоматический порог по методу Оцу.
   - **Triangle** – автоматический порог по методу треугольника.
   - **Adaptive Mean** – адаптивный порог по среднему арифметическому; параметры: размер окна (нечётное число, 3…201), константа C.
   - **Adaptive Gauss** – адаптивный порог по взвешенному Гауссу; параметры: размер окна, C.
   - **Niblack** – локальный порог по Ниблэку; параметры: размер окна, коэффициент k.
   - **Sauvola** – улучшенный метод Ниблэка; параметры: размер окна, k, R.
   - **ISODATA** – итеративный метод; параметр: начальный порог.
   - **Background Symmetry** – метод, основанный на симметрии гистограммы фона; параметр: excess (чувствительность).
   - **Row Adaptive** – построчный адаптивный порог; параметры: размер окна, k.

3. **Морфологическая обработка**  
   Применяется к бинарному изображению после порога (и после возможной инверсии):
   - **Closing factor** – относительный размер ядра закрытия (0…0.10 от минимальной стороны изображения).
   - **Opening factor** – относительный размер ядра открытия.
   - **Kernel shape** – форма ядра: Rectangle, Ellipse, Cross.

4. **Инверсия маски**  
   Переключатель «Invert mask» – инвертирует чёрные и белые области.

5. **Режим отрисовки объектов**  
   Определяет, какие геометрические примитивы будут выделены и сохранены:
   - **None** – объекты не выделяются.
   - **Segmentation (Polygon)** – контуры объектов (полигоны).
   - **Bounding Box (Detect)** – ограничивающие прямоугольники (детекция).
   - **OBB (Oriented Box)** – ориентированные прямоугольники (минимальный повёрнутый прямоугольник).  
   Дополнительно: **Use convex hull** – строить выпуклую оболочку для контуров/прямоугольников.

6. **Визуализация (4 окна)**  
   - **Оригинал** – исходное цветное изображение.
   - **Бинарное** – результат бинаризации (чёрно-белое).
   - **Морфология** – результат после морфологических операций.
   - **Аннотированное** – исходное изображение с нарисованными объектами (цвет и толщина линий настраиваются в глобальных настройках).  
   Каждое окно поддерживает:
     - Зум: колёсико мыши.
     - Панорамирование: перетаскивание левой кнопкой.
     - Сброс зума: кнопка «Reset zoom in all views».

7. **Список объектов и координаты**  
   - Справа отображается список найденных объектов с чекбоксами. Выбранные объекты подсвечиваются на аннотированном изображении.
   - В текстовом поле ниже выводятся координаты выбранных объектов (в пикселях для bounding box, нормализованные для OBB/полигонов).

8. **Сохранение аннотаций**  
   Кнопка «Save Labels» сохраняет текущие объекты в YOLO‑формате в файл `.txt` рядом с изображением. Поддерживаются типы:
   - `detect` – class cx cy w h (нормализованные).
   - `obb` – class x1 y1 x2 y2 x3 y3 x4 y4 (нормализованные углы).
   - `segment` – class x1 y1 x2 y2 … (полигон).

9. **Пресеты**  
   - Выпадающий список пресетов (можно выбрать сохранённую конфигурацию).
   - «Сохранить как пресет» – запоминает текущие настройки (метод, параметры, морфологию, инверсию, режим отрисовки, hull).
   - «Удалить пресет» – удаляет выбранный.

10. **Автоподбор параметров (Auto Research)**  
    - Включение галочкой «Enable auto research».
    - Кнопка «Configure auto research» – открывает диалог, где можно для каждого метода задать:
      - включён/выключен;
      - диапазоны параметров (начало, конец, шаг);
      - для морфологии: инверсия, закрытие, открытие (с шагом).
    - Кнопка «Запустить автоподбор» – выполняет перебор всех комбинаций для текущего изображения или всех загруженных (опция «Search all images»).
    - Результат сохраняется в файл `_full_report.txt` в папке изображения, где для каждой комбинации записано количество найденных объектов и их координаты.

11. **Гистограмма**  
    Показывается распределение яркостей текущего изображения. Для методов, возвращающих числовой порог (Simple Threshold, Otsu, Triangle, ISODATA, Background Symmetry), на гистограмме отображается вертикальная красная линия порога.

12. **Лог выполнения**  
    Текстовое окно с сообщениями о загрузке, ошибках, применении пресетов и т.д. Можно скрыть/показать кнопкой «Показать лог».

### Работа с аннотациями (интерактивное редактирование)

При загрузке изображений автоматически загружаются существующие `.txt`‑файлы (если есть). Список объектов обновляется.  
**Внимание:** вкладка «Пороговые методы» **не** поддерживает интерактивное рисование/редактирование аннотаций (это доступно на других вкладках, например «Интерактивные методы»). Здесь объекты формируются автоматически на основе бинаризации и морфологии.

### Глобальные настройки

Цвета линий, толщина, масштаб шрифта и другие параметры отображения берутся из глобального менеджера `settings` (меню «Настройки» в главном окне). При изменении настроек вид аннотаций обновляется автоматически.
