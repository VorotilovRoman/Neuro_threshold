# viewing_dataset.md

## Вкладка «Просмотр аннотаций»

Вкладка предназначена для визуального просмотра и инспекции уже размеченных датасетов. Пользователь может загружать изображения вместе с их аннотациями (файлы `.txt` в YOLO‑формате), масками сегментации (например, для проверки ground truth), а также целые датасеты из YAML‑файлов. Поддерживается навигация по изображениям, отображение ограничивающих рамок (`detect`), ориентированных прямоугольников (`obb`), полигонов (`segment`), а также наложение цветных масок с регулируемой прозрачностью. Аннотации можно удалять, изменять класс объектов (через двойной клик) и сохранять обратно в файлы.

### Основные возможности

1. **Загрузка данных**
   - **Load Folder** / **Load Images** – стандартная загрузка изображений (см. `tools.md`). При этом аннотации автоматически загружаются из `.txt`‑файлов, лежащих рядом с изображениями.
   - **Load Labels** – дополнительная загрузка аннотаций из выбранной папки (файлы `.txt` с именами, совпадающими с именами изображений). Полезно, если аннотации хранятся отдельно.
   - **Load Masks** – загрузка масок сегментации (бинарных или мультиклассовых PNG/JPEG) из выбранной папки. Маски должны иметь те же имена, что и изображения. Для каждого класса (кроме фона 0) автоматически генерируется цвет (по хэшу от ID класса). Прозрачность наложения регулируется слайдером.
   - **Load YAML** – загрузка полного датасета по YAML‑файлу (формат YOLO). Загружаются изображения, аннотации и маски (если они есть в папках `masks/`). Для масок также генерируются цвета.

2. **Отображение**
   - **Изображение** – центральная область (`SmartGraphicsView`). Поддерживается зум (колёсико мыши) и панорамирование (левая кнопка). Сброс зума – кнопка «Reset zoom» (общая для всех вкладок, не показана на скриншоте, но присутствует в коде).
   - **Аннотации** – отображаются поверх изображения. Типы:
     - `detect` – прямоугольник (цвет из глобальных настроек `annotation`).
     - `obb` – ориентированный прямоугольник (4 точки).
     - `segment` – полигон (произвольное число точек).
   - **Выделение выбранного объекта** – цвет `selected` (из глобальных настроек).
   - **Маска** – накладывается полупрозрачным цветным слоем. Каждый класс (ненулевая метка) получает свой цвет. Прозрачность регулируется слайдером (0…100%).

3. **Управление объектами**
   - **Список объектов** – справа (группа «Bounding boxes»). Каждый элемент отображает индекс, класс и тип объекта (с координатами для `detect` или сокращённо для `obb`/`segment`).
   - **Выбор объекта** – клик по элементу списка подсвечивает объект на изображении (и меняет цвет линии).
   - **Удаление выбранного** – кнопка «Delete Selected» или контекстное меню (правый клик по объекту в списке → Delete). После удаления список обновляется.
   - **Изменение класса** – двойной клик по объекту в списке открывает диалог ввода нового целочисленного ID класса (≥0). Применяется ко всем типам аннотаций.

4. **Сохранение аннотаций**
   - **Save Labels** – сохраняет текущие аннотации (после возможных удалений или изменений классов) в файл `.txt` рядом с изображением. Используется универсальная функция `save_annotations`.

5. **Лог и настройки**
   - **Лог** – выводит сообщения о загрузке, сохранении, ошибках. Кнопка «Показать лог» скрывает/показывает окно лога.
   - **Глобальные настройки** – цвета линий и толщина (`settings`) влияют на отображение аннотаций. При изменении настроек изображение перерисовывается.

### Рабочий процесс

1. Загрузить изображения (папка, файлы или YAML).
2. (Опционально) Загрузить маски для визуального сравнения.
3. Просматривать изображения с помощью кнопок навигации.
4. При обнаружении ошибки в разметке:
   - Выбрать объект в списке или на изображении (клик).
   - Удалить его (Delete Selected) или изменить класс (двойной клик).
5. После редактирования сохранить изменения (Save Labels).
6. При необходимости загрузить другой YAML‑датасет или переключить папку.

### Особенности реализации

- **Унифицированные аннотации** – все объекты хранятся в виде кортежей (`'detect'`, `'obb'`, `'segment'`), как и в других вкладках. Функция `load_annotations` автоматически определяет тип.
- **Маски** загружаются функцией `load_dataset_from_yaml_with_masks` (из `image_io.py`) или отдельно через папку. Маски масштабируются до размера текущего изображения (если не совпадают). Для генерации цветов используется HSV‑палитра (оттенок = `cls * 37 % 180`), что даёт разные цвета для разных классов.
- **SmartGraphicsView** используется только для отображения, без интерактивного редактирования геометрии (в отличие от вкладки «Добавить аннотации»). Редактирование ограничено удалением и сменой класса.
- При загрузке через YAML аннотации и маски извлекаются из структуры датасета (стандартная YOLO‑структура: `images/train`, `labels/train`, `masks/train`). Если маски не найдены – `loaded_masks[i] = None`.
- При изменении режима ресайза (чекбокс «Resize to max 1024px») программа предлагает перезагрузить изображения (с потерей несохранённых изменений). Отказ приводит к очистке текущих данных.

### Отличия от вкладки «Добавить аннотации»

| Функция | «Добавить аннотации» (Labeler) | «Просмотр аннотаций» (ViewingDataset) |
|---------|-------------------------------|---------------------------------------|
| Рисование новых объектов | ✅ (прямоугольники detect) | ❌ |
| Редактирование геометрии | ✅ (перемещение, изменение размера, вращение OBB, точки полигона) | ❌ |
| Удаление объектов | ✅ | ✅ |
| Изменение класса | ✅ (двойной клик) | ✅ (двойной клик) |
| Отображение масок | ❌ | ✅ (с регулировкой прозрачности) |
| Загрузка YAML с масками | ❌ (только изображения и аннотации) | ✅ |
| Сохранение аннотаций | ✅ | ✅ |

Таким образом, вкладка «Просмотр аннотаций» ориентирована на **инспекцию и лёгкую коррекцию** уже готовых датасетов, в то время как «Добавить аннотации» – на **создание разметки с нуля**.

### Связь с общими инструментами

- Навигация, загрузка, лог, `SmartGraphicsView`, сохранение аннотаций – см. `tools.md`.
- Функции `load_annotations`, `save_annotations`, `load_images_universal`, `load_dataset_from_yaml_with_masks` – из `utils.image_io`.
- Глобальные настройки цвета и толщины (`settings`) влияют на отображение аннотаций.

### Примечания

- Маски должны быть в формате PNG/JPEG с одноканальными значениями (0 – фон, 1..N – классы). Если маска цветная, используется первый канал.
- При загрузке YAML‑датасета путь к маскам ищется в подпапке `masks/` относительно корня датасета. Поддерживаются структуры `masks/train`, `masks/val`, `masks/test`, а также `masks/` на одном уровне с `images/`.
- Для очень больших датасетов (тысячи изображений) загрузка всех масок может занять время; прогресс‑диалог не предусмотрен, но загрузка выполняется в основном потоке.
