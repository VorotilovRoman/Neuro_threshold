# yolo_find_img.md

## Вкладка «Сортировка с YOLO»

Вкладка предназначена для автоматического поиска изображений в заданной папке, на которых модель YOLO (детекция) обнаруживает объекты указанного класса. Пользователь выбирает корневую папку, указывает модель YOLO, задаёт целевой класс (ID), и программа рекурсивно сканирует все изображения, выполняет инференс и отображает файлы, в которых найден искомый объект. Результаты можно просматривать в виде списка или миниатюр, сортировать по уверенности, а также копировать или перемещать отобранные файлы в другую папку. Поддерживается отображение дерева файлов для быстрой навигации.

### Основные возможности

#### 1. Выбор папки и навигация по файлам
- **Кнопка «Выбрать папку»** – задаёт корневую директорию для рекурсивного поиска.
- **Дерево файлов** (левая панель) – отображает структуру выбранной папки с фильтром по расширениям изображений (`.jpg`, `.jpeg`, `.png`, `.bmp`, `.tif`, `.tiff`, `.webp`). Двойной клик по любому файлу открывает его в стандартном просмотрщике ОС.

#### 2. Настройка модели YOLO
- Виджет **Параметры YOLO** (аналогичен `YOLOInferenceSettings`):
  - **Модель** – выбор файла `.pt` (кнопка «Обзор»).
  - **Порог уверенности** (0.01–1.0) – минимальная уверенность для детекции.
  - **Порог IoU (NMS)** (0.01–1.0).
  - **Размер входа** (32–1280).
- **Устройство** – выпадающий список: `auto`, `0` (CUDA), `cpu`, `mps`. `auto` выбирает CUDA при наличии.
- При выборе модели автоматически загружается и отображается **список доступных классов** (справа в группе «Целевая метка»). Это помогает убедиться, что целевой ID корректен.

#### 3. Задание цели поиска
- **Class ID** – числовой идентификатор искомого класса (по умолчанию 0).
- **Список доступных классов** – отображает все классы, которые умеет распознавать загруженная модель (в формате `ID: имя`). Обновляется при выборе новой модели.

#### 4. Сканирование папки
- Кнопка **«Начать поиск»** – запускает рекурсивный обход всех изображений в выбранной папке.
- Процесс выполняется в отдельном потоке (`ScanThread`) с отображением прогресса в процентах и количеством обработанных файлов.
- Для каждого изображения модель YOLO выдаёт предсказания. Если найден хотя бы один объект с целевым class ID, файл добавляется в список результатов вместе с максимальной уверенностью среди всех обнаруженных объектов этого класса.
- Кнопка **«Стоп»** – прерывает сканирование (с сохранением уже найденных результатов).
- По окончании сканирования автоматически открывается **режим миниатюр** (если есть результаты), и выводится сообщение с количеством найденных файлов.

#### 5. Отображение результатов
Результаты отображаются в правой панели (группа «Найденные файлы (содержат метку)»). Доступны два режима просмотра:

##### Режим «список» (по умолчанию)
- Элементы списка: имя файла и максимальная уверенность (например, `image.jpg (conf: 0.873)`).
- Каждый элемент имеет чекбокс для выбора.
- Двойной клик по элементу открывает файл в системном просмотрщике.
- Кнопка **«Сортировать по убыванию уверенности»** переупорядочивает список (и обновляет миниатюры, если активен режим миниатюр).

##### Режим «миниатюры»
- Переключение кнопкой **«Режим: миниатюры»** (работает только после завершения сканирования).
- Каждая миниатюра – это уменьшенное изображение (максимальная сторона 150 пикселей) с чекбоксом, именем файла и уверенностью.
- **Двойной клик по миниатюре** (или по подписи) открывает оригинальный файл.
- Миниатюры кэшируются во временной папке `temp_thumbs` (создаётся рядом со скриптом) для ускорения повторного отображения. При запуске программы временная папка очищается.
- Чекбоксы в миниатюрах синхронизированы с чекбоксами в списке (и наоборот).
- **Чекбокс «All»** – выделяет/снимает выделение всех файлов (работает в обоих режимах).

#### 6. Копирование / перемещение выбранных файлов
- **Целевая папка** – задаётся кнопкой «Обзор» и отображается в поле.
- **Копировать** – копирует все выбранные файлы в целевую папку (сохраняя оригиналы).
- **Переместить** – перемещает выбранные файлы (удаляя их из исходной папки). После перемещения результаты обновляются: удалённые файлы исчезают из списка и миниатюр.
- При копировании/перемещении ведётся лог с указанием успешных операций и ошибок.

#### 7. Лог выполнения
- Внизу правой панели расположен лог-виджет (`LogWidget`), в который выводятся все сообщения: выбор папки, загрузка модели, прогресс сканирования, ошибки, результаты копирования.
- Лог можно очистить кнопкой «Очистить лог».

### Рабочий процесс

1. **Выбрать папку** – корневая директория для поиска.
2. **Выбрать модель YOLO** – указать `.pt` файл. Дождаться загрузки списка классов.
3. **Установить целевой Class ID** (например, 0 для «person»).
4. При необходимости подкорректировать пороги и размер входа.
5. Нажать **«Начать поиск»**.
6. Дождаться завершения сканирования (прогресс-бар и лог показывают ход).
7. Просмотреть результаты в списке или миниатюрах.
8. Отметить нужные файлы чекбоксами (или «All»).
9. Выбрать целевую папку и нажать **«Копировать»** или **«Переместить»**.
10. При необходимости отсортировать результаты по уверенности (кнопка сортировки).

### Особенности реализации

- **Поток сканирования** – `ScanThread` наследует `QThread`. Он рекурсивно обходит все подпапки, загружает изображения через `read_image_with_fallback_find` (поддержка повреждённых файлов и разных форматов), выполняет инференс YOLO и проверяет наличие целевого класса.
- **Кэш миниатюр** – миниатюры сохраняются во временную папку с именем, основанным на MD5‑хеше пути к файлу. При повторном открытии той же папки (или после сортировки) миниатюры загружаются из кэша, что ускоряет переключение режимов.
- **Синхронизация выбора** – между списком и миниатюрами поддерживается двусторонняя синхронизация состояний чекбоксов через сигналы.
- **Открытие файлов** – используется `QDesktopServices.openUrl`, что открывает файл в программе, ассоциированной с его типом в ОС.
- **Сортировка** – выполняется in‑place над списком `self.result_file_paths`, после чего обновляются оба представления (список и миниатюры).

### Зависимости

- `ultralytics` – для загрузки модели YOLO и инференса.
- PyTorch, OpenCV, NumPy – стандартные.
- `shutil`, `hashlib`, `os`, `sys` – стандартные библиотеки Python.

### Примечания

- Вкладка работает **только с детекцией** (модель должна быть детекционной, а не сегментационной). Если модель сегментационная, предсказания будут некорректны.
- При сканировании больших папок (тысячи изображений) процесс может занять длительное время. Прогресс-бар и кнопка «Стоп» позволяют прервать операцию.
- Миниатюры генерируются синхронно после окончания сканирования (при переключении в режим миниатюр). Для большого количества файлов это может вызвать кратковременную задержку, но она компенсируется кэшированием.
- При перемещении файлов физическое удаление происходит немедленно; убедитесь, что выбрана правильная целевая папка.
- Временная папка `temp_thumbs` очищается при закрытии программы.

### Связь с общими инструментами

- `LogWidget` – стандартный (см. `tools.md`).
- `YOLOInferenceSettings` – используется для настройки параметров модели.
- `read_image_with_fallback_find` – из `utils.image_io`.
- Класс `ClickableLabel` – кастомный QLabel для обработки двойного клика по миниатюре.
